# High Fidelity prototype

Het HiFi prototype is wat er in de ontwikkelingsfase zal worden ontwikkeld tot website. Het is dus belangrijk
dat deze goed, compleet en duidelijk is.

Ik heb het HiFi prototype gemaakt in Figma. Figma is een web programma voor ontwerp zoals Sketch en Adobe XD.
Het is bijna exact hetzelfde als wat er op de website staat, met uitzondering van een CSS grid dingetje in
de tijdlijn.

Om het responsive UI duidelijk te krijgen voor de ontwikkelaar heb ik alle pagina's gemaakt voor een 13" Macbook,
een iPhone 16 en een iPhone 8/SE. De iPhone 16 is breed genoeg om de header volledig weer te kunnen geven,
maar de iPhone 8 vereist een hamburger menu. De Macbook is zo groot dat de pagina inhoud gecentreerd is.

<iframe src="/2425/lmfehres/hifi-page.pdf" style="width: 100%; height: 800px;"/>

Figma ondersteunt components waardoor herhaalde UI elementen kunnen worden gesynchroniseerd met alle plekken
waar ze worden gebruikt. Als een component wordt gekopieerd kan de tekst ervan wel woorden aangepast zonder
de andere instances aan te passen, dit is gebruikt voor de tijdlijn items en text input velden. Het text input
veld component is responsief, het heeft een minimumgrootte van hug (zo klein mogelijk zonder text wrapping) en
een standaardgrootte van fill (zo veel mogelijk horizontale ruimte gebruiken).

<iframe src="/2425/lmfehres/hifi-component.pdf" style="width: 100%; height: 200px;"/>

Met zo'n prototype kan niet alleen een user experience test worden uitgevoerd om te kijken of de website wel
overzichtelijk is, maar dit kan ook 1 op 1 worden gepubliceerd op de website. Figma heeft hier tools
zoals "dev mode" voor ontwikkelaars voor, helaas is dit niet beschikbaar zonder geld over te harken.
